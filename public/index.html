A low-level functional language

For exploring algebraic effects, safe shared mutability, and other novel features

Opt-out Move Semantics

move_value x = ()

// Most types are not shared and use move semantics by default
vec = Vec.of [1, 2, 3]
move_value vec  // ok!
move_value vec  // error! Vec already moved

// But shared types can be defined which enable cleaner code and a gentler learning curve.
shared type Expr =
    | Int I32
    | Var (name: String)
    | Add Expr Expr  // No explicit boxing since `Expr` is shared

foo = Expr.Var "foo"
move_value foo  // ok!
move_value foo  // ok!

Traits via Implicits

// Abilities in Ante are just sugar for types holding closure values
ability Foo t =
    foo: fn t -> U32

// Since they are just values, we can create them in normal constructor expressions
// and no global coherence or orphan restrictions are required
implicit ord_i32_descending = Ord I32 with
    cmp a b = Ordering.from_int (b - a)

// Ability values can be defined locally and closures can capture data,
// making implementing them on interned types much easier
implicit display_interned (ctx: InternCtx): Display MyInternedType with
    display interned =
        // Use `ctx` from the closure environment
        data = ctx.get interned
        display data

Effects

// Effects are just abilities with `resume` functions, enabling effect handlers
ability Throw err =
    throw: resume fn t -> Never

// And capabilities are just ability values, which are often passed as implicits
signup (name: String) (email: Email) {t: Throw ValidationErr} {Throw IOErr} {IO}: User =
    // Unlike monads, capabilities are trivial to combine: they are just implicit parameters
    cfg = read_cfg_from_disk () // Requires `Throw IOErr` and `IO`
    // And they can be passed explicitly if desired (or required)
    validate name email {t}

// No function coloring problem: map_vec trivially works with exceptions, async, etc.
// Any capabilities required by `f` are captured in its closure
map_vec (v: Vec t) (f: fn t => u): Vec u =
    Vec.of (map v f)

Safe Shared Mutability

var vec = Vec.of [1, 2, 3]

ref1 = mut vec  // Create a temporary mutable reference to vec
ref2 = mut vec

// Ok! It is safe to call `Vec.push` with shared mutable references
ref1.push 4
ref2.push 5

// Retrieving a reference to an element is also possible, but we can't
// use other mutable references to the same value while it is active.
elem = ref1.get 0

Generators

// Generators are implemented via effects
ability Emit elem =
    emit: resume fn elem -> Unit

// One advantage of generators is that they are easier to write than iterators.
// We'll supply the `Emit a` value but the caller needs to provide a capability for `Emit b`
map (stream: fn (Emit a) -> Unit) (f: fn a => b) {Emit b} =
    handler emit_a = Emit fn elem ->
        emit (f elem)
        resume ()
    in stream emit_a

// From: `a,b,c\n1,2,3\n4,5,6`
// Return: `[["1", "2", "3"], ["4", "5", "6"]]`
parse_csv (text: String): Vec (Vec String) =
    lines text |> skip 1 |> map (fn line -> split line "," |> Vec.of) |> Vec.of

Platform Agnostic

// `main` takes any required libraries and primitives as arguments
main {Linux} = // fork, execve, etc
main {Wasm}  = // __wasi_fd_read, __wasi_fd_write, etc

// But you are encouraged to use the IO module which is available on most common platforms
main {IO} = ...

// Libraries may still defer to more efficient implementations when needed
my_async_io {IO} =
    if IO.linux () is Some api
    then // use io_uring
    else // fallback

// Libraries requiring code to be linked in must specify this. Otherwise, they
// would break on exotic targets like Wasm or the Beam VM which can't support dynlibs
compile_with_llvm_dynlib (llvm_bitcode: String) {Llvm} = ...

Goals

Ante aims to help bridge the gap between low-level languages like C++/Rust and higher level garbage-collected languages like Java, OCaml, or Haskell. Ante aims to make developing programs easier by allowing patterns like shared mutability which allow for more design flexibility while still remaining thread safe and memory safe.

Safe Shared Mutability

Shared mutability is a common cause of bugs in other languages. In Ante, not only is it safe, it is zero-cost. The type system will prevent unsafe operations on mutably shared references at compile-time.

Secure

Ante has capability-based security. If you don’t pass an IO object to a function, it can’t use it. It cannot even use external linked-in code without being allowed to do so.

Abilities

Abilities combine traits and an effect system and provide great code reuse. They are easier to use when compared to monads and provide clearer type errors when things are not as expected.

Flexible Compilation

Compared to similar compilers, Ante’s compiler is both incremental and does not require monomorphization in debug builds.

High & Low Level language

To try to bridge the gap between high and low level languages, ante has high-level abstractions implemented in userland by a low-level core.

Easier Lifetime Variables

Instead of abstract or numbered names, Ante’s lifetime variables are named after the variable(s) they borrow, making error messages more clear
